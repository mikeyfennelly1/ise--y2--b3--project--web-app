= ise--y2--b3--project--web-app

Spring Boot web application that subscribes to a NATS message broker, persists received messages to PostgreSQL, and exposes a REST API for managing subscriptions and querying time-series data.

:toc:
:toc-title: Contents
:toclevels: 2

== Starting the application

[source,bash]
----
./start.sh
----

`start.sh` waits for PostgreSQL to become available, then starts the Spring Boot application via `./gradlew bootRun`.
Required environment variables (`POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `WEB_APP_PORT`) are sourced from a `.env` file in the parent directory.

== Versioning

This project uses https://semver.org[semantic versioning] (`MAJOR.MINOR.PATCH`).
The current version is defined in `app/build.gradle`.

== Architecture

The application is a single Spring Boot service split into two subsystems.

=== Consumer subsystem (`/api/consumer`)

Manages a live subscription to a NATS message broker.
Subscriptions are organised as a *hierarchical topic tree* — each node in the tree represents a stream, and children of a node represent sub-subjects (e.g. `device` → `device.sysinfo`).

Users can modify the topic tree at runtime:

* **Constructively** — add a new root stream or attach a child stream beneath an existing node.
* **Destructively** — remove a stream (and its subtree) from the tree.

All messages received on any subscribed topic are assumed to share the same payload format and are handled uniformly.
Received messages are persisted to a PostgreSQL time-series table via the consumer repository.

=== Reporting subsystem (`/api/reporting`)

Reads from the persisted time-series data in PostgreSQL and exposes it for querying and reporting.
